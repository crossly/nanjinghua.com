# 常见设备录音技术指南

目标是取得人声清楚、来源关系完整的设备原始文件。设备能力有限并不会使录音失效；伪造高规格、只留下聊天软件压缩件或覆盖原件才会破坏档案价值。

## 环境

1. 选有软家具、窗帘或衣物的安静小房间，避开空房间、厨房和楼道。
2. 关窗，暂停风扇、空调强风、电视和背景音乐；请在场的人停止交谈。
3. 手机开飞行模式或关闭通话、闹钟和应用提示音。
4. 先安静录 10 秒，再录一段正常说话，共试录 20 秒并回放。
5. 听到持续电流声、施工声、明显回声或声音忽大忽小时，先换位置再正式录。

## 设备与格式

### 便携录音机或支持无损录音的设备

- 优先选择未压缩 PCM `WAV`，采样率 `48 kHz`，位深 `24 bit`。
- 一位说话者可以录单声道；设备固定输出立体声时保持设备原设置，不必现场转换。
- 关闭自动降噪、变声、混响、语音增强和背景音乐。
- 如果可看电平，让正常说话峰值大约落在 -12 至 -6 dBFS，不进入红色或 `CLIP`。

### 普通手机

- 使用系统录音应用的“无损”“高质量”或 `WAV` 选项；没有这些选项时，使用应用能直接导出的最高质量格式。
- 不要为了写成 48 kHz / 24 bit 而把低规格文件升采样或转格式。手机导出的文件就是原始文件，原样保留并如实记录格式。
- 不从微信、QQ 或其他聊天软件的语音消息功能录制，也不把聊天软件转发后的文件当原件。
- 保证储存空间充足、电量高于 30%，并避免手持摩擦设备。

## 距离与摆放

- 设备麦克风距嘴约 **15 至 20 厘米**，放在稳固桌面或支架上，略偏向嘴的一侧，减少喷麦。
- 不把设备放在会震动的空桌面中央；可以垫一块折叠软布，但不要盖住麦克风孔。
- 采集者与参与者分坐麦克风两侧；访谈时不要移动设备。
- 能接耳机时由采集者监听；不能监听时，每个主要环节结束后回放片段。

## 文件命名

文件名只用参与者代码，不用姓名、手机号、出生年月日或地址。

格式：`参与者代码_采集日期_当日序号.扩展名`

示例：`NJH-S01_20260717_01.wav`

- 参与者代码：项目预先分配，如 `NJH-S01`。
- 采集日期：八位数字 `YYYYMMDD`。
- 当日序号：两位数字，从 `01` 开始。
- 设备自动文件名先原样导出；完成第一次校验后复制一份按规则命名，不在设备唯一副本上操作。清单同时记录设备原名与交接文件名。

## 原始文件保留

1. 用数据线、隔空传送的“保留原文件”方式或存储卡直接导出，不经聊天软件转发。
2. 第一次导出后立即计算 SHA-256；把值写入交接清单。
3. 保留设备输出状态的**原始文件**，不在上面裁剪、降噪、改音量、改格式或覆盖保存。
4. 至少保留两份经过校验的副本，其中一份位于不同的加密存储介质或项目约定存储区。
5. 转写、裁剪隐私片段、降噪或转码都从原件复制生成**派生文件**，记录使用的软件、版本、参数、日期和操作人。
6. 派生文件的记录同时写入原件 SHA-256，使其始终能追溯到原件。

## 现场故障处理

- **爆音或削波**：设备离远 5 厘米或降低输入增益，重新试录；原失败文件仍先保留并在清单标“未采用”。
- **底噪很大**：先停设备和电器，再换远离窗户、硬墙的位置；不要以强降噪代替重录。
- **中途来电或中断**：停止后新建下一序号，不在设备上拼接。
- **说出隐私信息**：不修改原件；记录开始和结束时间，交接时标明派生发布版必须裁剪。
- **只得到压缩格式**：原样保存并记录实际格式，不转换后冒充 WAV 原件；条件允许时重新录。
