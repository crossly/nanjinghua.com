# 无真人素材流程演练

演练日期：2026-07-17  
交接编号：`HANDOFF-20260717-SIM`  
参与者代码：`SIM-0001`

本目录验证“采集 → 授权 → 分区 → 校验 → 派生 → 公开候选”的文件流。所有身份和元数据均为虚构，音频是 1 秒数字静音，不包含真人声音。

## 演练步骤

1. 用虚构参与者“测试参与者甲”建立受限身份对应文件，放入独立 `restricted/identity/` 分区。
2. 用独立授权凭证记录公开播放“同意”、公开下载“不同意”、研究复用“同意”，放入独立 `restricted/consent/` 分区。
3. 生成 48 kHz / 24 bit 单声道 PCM WAV 静音原件。
4. 从原件生成 MP3 派生文件，模拟浏览器发布转码；处理记录保留原件 SHA-256。
5. 公开候选 JSON 只写化名和概括背景，不含真实身份、联系信息或授权扫描。
6. 对五件交接文件计算 SHA-256，并用清单逐件复核；空派生目录场景另由自动化测试验证。
7. 模拟身份与授权分别写入两件离线受限存储、清除在线传送临时副本；本仓库只保留明确标为虚构的演练文件。

## 分区结果

- `restricted/identity/SIM-0001-identity.txt`：受限身份分区，虚构。
- `restricted/consent/SIM-0001-consent.txt`：受限授权分区，虚构。
- `originals/SIM-0001-original.wav`：原始文件目录，数字静音。
- `derivatives/SIM-0001-public.mp3`：派生文件目录，数字静音。
- `public/SIM-0001-metadata.json`：公开候选目录，假数据。
- `manifest.sha256`：上述五件文件的独立校验值。

## 演练结论

身份与授权可以分别进入不同权限分区和离线存储，公开候选元数据不泄露受限字段；原件和派生件是不同文件，派生处理可追到原件校验值。此演练只证明工作流可执行，不代表已经取得 Issue T10 所需的真人录音。
