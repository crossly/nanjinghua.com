# 安全交接清单

采集者和项目接收编辑各核对一次。公开表单只用于表达录音意愿，不接收录音、身份或授权文件。

## 1. 建立交接编号

- [ ] 项目分配交接编号：`HANDOFF-YYYYMMDD-__`
- [ ] 项目分配参与者代码：`NJH-S__`
- [ ] 文件和目录中没有真实姓名、手机号、身份证号或精确住址。

## 2. 分开整理

在加密存储区建立以下目录；实际交接时分别限制访问，不把它们压成一个无密码附件：

```text
HANDOFF-YYYYMMDD-__/
├── 01-originals/          原始录音，只读
├── 02-consent/            授权凭证，受限
├── 03-identity/           真实身份与代码对应，受限且与授权分开
├── 04-public-metadata/    化名和概括背景，公开候选
├── 05-derivatives/        裁剪或转码文件及处理记录
└── manifest.sha256        文件校验值
```

- [ ] `01-originals/` 来自设备直接导出，不是聊天软件语音或转发件。
- [ ] `02-consent/` 每项授权均明确选择，没有空白项或预先勾选。
- [ ] `03-identity/` 只含处理授权和撤回所需的最少身份联系资料。
- [ ] `04-public-metadata/` 不含真实姓名、签名、联系方式、精确地址或授权扫描件。
- [ ] `05-derivatives/` 每件文件记录来源原件 SHA-256、软件、版本、参数、日期和操作人。

## 3. 核对成年人和授权

- [ ] 授权书确认参与者录音当天已满 18 周岁。
- [ ] 公开播放：□ 同意　□ 不同意
- [ ] 公开下载：□ 同意　□ 不同意
- [ ] 研究复用：□ 同意　□ 不同意
- [ ] 采集者没有因某项不同意而劝说参与者改选。
- [ ] 授权书版本和签署日期完整，参与者已取得副本。

## 4. 核对背景和录音

- [ ] 背景只含出生年代、成长片区、长期居住、家庭语言和普通话使用概况。
- [ ] 共同词句完整，或已经列出缺项原因。
- [ ] 至少有一段自然讲述，或已经列出停止原因。
- [ ] 发现第三人姓名、联系方式、详细地址等隐私片段时，已记录原件时间点并标记派生版裁剪，未修改原件。
- [ ] 设备、应用、格式、采样率、位深或实际可取得的信息已经记录；未知项写“未知”，不猜测。

## 5. 计算和复核 SHA-256

在 macOS 或 Linux 上，对交接目录内文件逐件计算：

```bash
shasum -a 256 01-originals/* 02-consent/* 03-identity/* 04-public-metadata/* 05-derivatives/* > manifest.sha256
```

接收端在隔离目录中验证：

```bash
shasum -a 256 -c manifest.sha256
```

- [ ] 采集端已经保存清单。
- [ ] 接收端验证全部为 `OK`；不一致时停止导入，重新传送受影响文件。

## 6. 传送

- [ ] 使用项目约定的端到端加密文件传送、加密移动介质或加密临时存储链接。
- [ ] 解密密码通过与下载地址不同的渠道发送。
- [ ] 不使用公开网盘链接、普通电子邮件附件、微信群文件或聊天软件语音消息。
- [ ] 临时链接设置最短可行有效期，接收确认后撤销。
- [ ] 双方只向本次确需处理的人员开放相应目录。

## 7. 接收和收尾

- [ ] 接收编辑分别记录原始文件、授权凭证、身份对应和公开元数据的存放位置与访问权限。
- [ ] 接收编辑把三项授权录入处理记录，并由第二人或另一次复核确认。
- [ ] 原件至少有两份校验通过的受限备份；派生处理不覆盖原件。
- [ ] 项目向采集者发出“已接收并通过校验”确认，列明交接编号和文件数。
- [ ] 采集者收到确认、核对自己应保留的授权副本和备份责任后，删除下载目录、聊天缓存和其他临时工作副本。
- [ ] 任何错发、丢失、权限配置错误或未经授权访问立即按隐私或安全请求上报，不继续转发排查材料。
