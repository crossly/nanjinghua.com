## Problem Statement

希望访问、了解或研究南京话的人，目前很难找到一个面向公众、结构清晰且可持续核查的数字档案。网络内容经常混用当代南京主城区地方话、南京市域内其他方言以及历史文献中的“南京音”“南京官话”，把转载次数当作证据强度，或用“正宗”“唯一正统”等结论掩盖语言变化与学术分歧。历史材料、语音、口述记忆、研究观点和民间说法也常被放在同一层级，读者无法判断材料是什么、来自哪里、何时形成、能否公开复用。

项目拥有 `nanjinghua.com` 域名，希望建立一个名为“南京话”、完整说明为“南京话的历史”的公共数字档案。它需要既能让普通公众轻松进入，又能让研究者追到完整引用；既能传播南京话文化，又不以宣传替代证据；既能保存真人声音和文化记忆，又能尊重说话者隐私、版权与撤回权。

## Solution

建设一个证据优先、面向公众、无需登录即可访问的南京话历史数字档案。首发以“南京话是什么？”为唯一完整专题集合，通过六篇职责清晰的专题文章、20 至 30 个正式档案条目，以及 3 至 5 位成年说话者贡献的 12 至 20 个原创语音样本，解释南京话的地域边界、材料来源、历史记录、当代变化和相关文化形式。

网站将档案条目、专题文章和专题集合分层组织。每件正式档案分配永久档案编号，记录证据身份、权利状态、档案时间、档案地点、引用记录及修订历史。原始材料、研究观点、口述记忆和待考说法可以并存，但不得被表达成同等性质的确定史实。界面以简体中文服务普通公众，同时保留外文来源原题名、关键原文与中文说明。

系统使用 TanStack Start、React、TypeScript 和 TanStack Router，部署到 Cloudflare。Git 仓库是正式档案与专题内容的权威来源，R2 保存授权媒体，D1 仅保存公众线索与处理状态，Turnstile 保护提交入口。网站优先预渲染公开内容，并以中国大陆稳定访问、无障碍、长期引用和可迁移性作为发布门槛。

## User Stories

1. As a 初次访问者, I want 一进入网站就看到“南京话”及“南京话的历史”, so that 我能立即理解网站研究的对象。
2. As a 初次访问者, I want 从“南京话是什么？”导览开始浏览, so that 我不需要先学会使用档案数据库。
3. As a 普通读者, I want 看到南京话、南京地区方言和历史南京语音的明确区别, so that 我不会把不同对象误认为同一种语言形式。
4. As a 普通读者, I want 知道网站是独立项目而非政府或学术机构, so that 我能正确判断其责任与权威范围。
5. As a 南京本地读者, I want 从历史脉络进入真实材料, so that 我能理解熟悉语言背后的时间变化。
6. As a 离乡南京人, I want 听到带背景说明的南京话语音, so that 我能接触真实声音而不是表演化配音。
7. As a 文化爱好者, I want 浏览与南京话有明确关系的白局、童谣、俗语、曲艺和方言书写, so that 我能理解语言如何存在于文化表达中。
8. As a 文化爱好者, I want 网站排除与南京话无关的泛城市史、旅游和美食内容, so that 档案不会失去主题边界。
9. As a 普通读者, I want 每项历史性陈述显示其证据身份, so that 我能区分原始材料、研究观点、口述记忆和待考说法。
10. As a 普通读者, I want 学术分歧以观点和依据并列呈现, so that 我不会被虚假的统一结论误导。
11. As a 普通读者, I want 无出处的流行说法被标为待考, so that 多次转载不会被包装成史实。
12. As a 普通读者, I want 明确看到历史南京语音与当代南京话不能直接画等号, so that 我不会把“南京官话”等概念误解为今天的标准答案。
13. As a 普通读者, I want 历史叙事从可核查的最早证据开始, so that 网站不会虚构南京话起源于某个朝代。
14. As a 普通读者, I want 更早的城市与语言背景被标为背景而非连续性证明, so that 朝代更替不会替代语言证据。
15. As a 普通读者, I want 专题文章只回答一个清晰问题, so that 长篇历史不会混成无法核查的概述。
16. As a 普通读者, I want 从专题文章的关键论断跳转到档案条目或外部文献, so that 我可以自行核查来源。
17. As a 档案浏览者, I want 按内容类型筛选, so that 我可以分别浏览档案条目、专题文章和专题集合。

## Implementation Decisions

- 网站主名称为“南京话”，完整说明为“南京话的历史”，域名为 `nanjinghua.com`。它是独立公共文化档案，不使用暗示政府、博物馆、研究院或专家机构身份的命名。
- 核心对象是南京主城区地方话；南京市域内其他地方话作为南京地区方言关联收录；历史南京语音单独建模，不与当代南京话直接等同。
- “正本清源”是追溯来源、区分证据和呈现争议的编辑方法，不是预设唯一正统的品牌口号。
- 只收录与南京话存在可说明关系的文化材料。白局、童谣、俗语、曲艺、方言书写和音像记录使用受控“文化形式”分类；泛南京城市史不进入档案。
- 内容模型分为档案条目、专题文章和专题集合。专题文章的关键论断引用档案条目或可核查外部文献；专题集合按主题组织二者。
- 证据身份固定为原始材料、研究观点、口述记忆和待考说法。证据身份与权利状态是两个独立字段。
- 首发只有一个完整专题集合“南京话是什么？”，包含 4 至 6 篇专题文章、20 至 30 个正式档案条目和 12 至 20 个原创语音样本，不创建空栏目。
- 六篇首发文章分别回答：南京话是什么；怎样知道其历史；历史南京音与南京官话是什么；南京话如何被记录；当代南京话怎样变化；南京话存在于哪些文化形式。
- 历史叙事不预设朝代起源。更早时期仅作有证据的背景；明清历史南京语音、晚清至民国记录、1949 年以后至当代材料分段研究，最终分期由来源决定。
- 全网检索用于广泛发现线索，但只有可追溯来源能够支撑结论。原始文献、地方志、字书、历史录音、学术论文与专著、大学及图书馆馆藏优先；政府文化机构和非遗机构材料辅助；新闻、自媒体、论坛、短视频与百科只作线索来源。
- 研究范围包含中文、英语、日语及其他可发现语言。外文引用保留原题名和关键原文并提供中文说明；机器翻译不能独立支撑关键结论。
- 引用记录包含责任者、题名、出版物、日期、页码或章节、稳定标识符、馆藏信息和访问日期。优先 DOI、ISBN、馆藏号和永久链接；允许的公版或授权文件保存原件和校验值；链接失效保留书目信息并寻找替代馆藏。
- 每个正式档案条目分配形如 NJH000001 的永久档案编号。编号不随标题、分类或证据身份变化，撤回后不复用；稳定网址以编号为核心。
- 档案时间区分材料日期、所述时期和有依据的推定时期。档案地点区分原载地名、当时行政归属、可确认的当代位置、说话者成长地点和采集地点。
- 语言变体是一等档案对象。多种读法关联年代、生活片区、说话者背景和使用场景，不标正确、错误、正宗或不正宗。
- 语音呈现采用录音、汉字转写和 IPA 三层。普通话拼音只辅助搜索；IPA 记录者和审核状态可见。
- 首批录音来自 3 至 5 位成年说话者，覆盖不同出生年代和成长片区，合计 12 至 20 个样本。每人录制共同词句和自然讲述，并公开采样局限，不进行人口代表性推断。
- 用户负责联系说话者并取得原始录音和授权；项目制作采集词表、访谈提纲、录音说明、分项授权书和提交模板，并负责后续整理、转写、IPA 初稿、元数据与页面。
- 语音授权分别控制公开播放、下载和研究复用。公开说话者背景使用化名与概括化信息；真实身份及授权凭证离线受限保存。首发不采集未成年人语音。
- 原始文件与派生文件分开保存，不覆盖原件；保存校验值、处理过程与来源关系。
- 事实修正和证据身份变化公开简要修订记录；撤回授权、隐私或人身风险允许移除媒体、目录或个人信息，权利保护优先于公开版本完整性。
- 文章公开作者、发布日期、最近修订日期和审核状态。审核状态固定为编辑核对后发布、待专家复核、专家复核；专家复核记录审核人、实际范围和日期。
- AI 可辅助资料发现、归纳和初稿，但不作为作者、专家或来源。页面披露 AI 辅助整理；未完成专家复核的 IPA 与争议结论如实标注状态。
- 正式档案和专题内容使用 Git 管理的结构化 Markdown 与元数据，并在构建时做模式校验。后续编辑团队扩大后可增加 CMS，但不改变权威内容模型。
- 内部元数据模型满足南京话档案所需专门字段；通用字段映射 Dublin Core，页面输出 Schema.org JSON-LD。首发不实现 IIIF、METS 或 MODS。
- 档案元数据采用 CC0；项目原创专题采用 CC BY 4.0；录音、扫描件、照片等媒体逐项服从自身权利状态。
- 正式专题、条目与元数据无需登录即可访问并允许搜索引擎收录。条目提供规范引用与单条结构化导出；批量数据集延后。
- 首页导览优先、搜索常驻，直接进入“南京话是什么？”，同时提供浏览全部档案入口；首页不是营销落地页。
- 固定检索维度为内容类型、证据身份、时间、地点、文化形式及是否包含可播放语音，另提供全文搜索，不开放任意自由标签。
- 导航、操作和史实说明使用简体中文现代汉语；南京话只在被展示和解释的内容中出现。首发不提供英文或繁体界面。
- 视觉方向是当代公共文化档案，优先真实文献、声音波形、旧地图、地名与人物照片，少量南京辨识度红色作强调；不采用泛黄宣纸、卷轴、城墙剪影或民国滤镜作为整站主题。
- 无障碍基线是 WCAG 2.2 AA。音频有文字转写和说明；地图与时间线有等价列表；扫描件有可读文本或摘要；支持键盘、清晰焦点、足够对比度和减少动态；状态不只依赖颜色。
- 应用采用 TanStack Start、React、TypeScript 与 TanStack Router。公开档案和专题尽量预渲染；动态表单与接口运行在 Cloudflare Workers。TanStack Query、Form 和 Table 只按实际需要引入。
- 正式档案及专题由 Git 仓库持有，授权媒体存入 R2，公众线索及状态存入 D1，Turnstile 保护表单。D1 不作为正式档案数据库。
- 首发不开放匿名文件上传。公众只提交说明、联系方式与材料链接；编辑确认来源和授权后再接收原件。
- 线索状态为已收到、待补充、核验中、已采纳、已关闭。已采纳只表示进入编辑工作，不等于原投稿直接发布。
- 联系方式与线索正文分离。已关闭或已采纳后 90 天删除联系方式；匿名化线索可保留用于防止重复调查；180 天无处理的线索触发提醒并在无法核验时关闭；权利、隐私或安全请求优先。
- 使用无 Cookie 的聚合式 Cloudflare Web Analytics，不使用广告、跨站追踪或用户画像，不记录可识别个人的站内搜索历史。
- Git 内容仓库保有远程副本和定期离线导出；R2 媒体有独立备份；D1 线索队列定期独立导出；撤回材料服从权利保留策略。
- 中国大陆稳定访问是发布硬门槛。页面不依赖 Google Fonts 等不稳定境外资源，并从多个大陆网络测试页面、搜索和音频。Cloudflare 表现持续不达标时允许增加符合备案要求的国内镜像或替代路径。
- 网站公开关于本站、编辑与证据原则、版权与开放许可、隐私政策、纠错撤回与权利申诉、无障碍说明、参与贡献与语音采集说明。
- 允许捐赠、文化项目资助和透明机构合作，但赞助不能修改结论、删除争议或购买权威背书。首发不投放干扰阅读的广告。
- 交付顺序为来源清单与多语种研究、内容模型、应用与内容并行制作、Cloudflare 服务接入、真人语音处理、全项验收和域名部署。录音到位前只提供预发布版本。
- 成本遵循低固定成本和付费前确认。首发优先 Cloudflare 免费或低用量服务，不购买商业字体、模板、CMS、分析工具或素材；文献、授权、专家、国内加速和备案服务需预先确认。

## Testing Decisions

- 测试只验证外部可观察行为，不断言 React 组件结构、TanStack 内部状态、私有函数或具体存储实现。
- 最高测试边界是一条运行在 Cloudflare 本地或预览环境上的真实浏览器流程：从首页导览进入专题文章，追到档案条目与引用记录，使用搜索和固定筛选，播放带转写的语音，并完成线索或纠错提交。
- 该浏览器流程在桌面和移动视口运行，覆盖正常、空结果、加载失败、媒体无下载许可、撤回占位、表单校验和 Turnstile 可替代测试路径。
- 浏览器验收包含键盘导航、焦点顺序、状态非颜色依赖、减少动态设置，以及自动化 WCAG 2.2 AA 检查；地图、时间线、音频和扫描件必须存在等价文本入口。
- 内容发布边界通过完整语料构建进行验证：缺少永久编号、证据身份、权利状态、档案时间、档案地点、引用记录、审核状态或授权字段时构建失败；编号重复或撤回编号复用时构建失败。
- 结构化导出以公开响应验证 Dublin Core 映射、Schema.org JSON-LD、稳定网址和规范引用，不测试序列化库内部实现。
- Worker HTTP 边界验证线索状态、输入校验、Turnstile、联系方式分离、数据最小化和错误响应；D1 或通知服务失败时不得丢失正式档案或产生公开内容。
- R2 媒体边界验证公开播放、禁止下载和允许研究复用等不同授权组合；原始文件不得通过公开网址暴露，派生文件必须能追溯来源。
- 修订与撤回使用公开页面行为验收：事实修订可见说明，权利撤回停止媒体展示，隐私删除不泄露历史个人信息，永久编号不被复用。
- 性能验收以正文优先、无境外阻塞依赖和移动网络可用为目标；发布前从中国大陆多个真实网络手动验证首页、搜索和音频，自动测试不能替代该检查。
- 首发内容验收要求 1 个完整专题集合、4 至 6 篇有署名专题、20 至 30 个完整档案条目、3 至 5 位成年说话者的 12 至 20 个授权语音样本、公开制度页面和无空栏目。
- 当前仓库尚无应用或测试先例。首个实现应建立上述最高层浏览器验收和内容构建门槛，不为每个视觉组件复制低层测试。

## Out of Scope

- 用户账号、评论区、论坛、用户直接发布和匿名文件上传。
- 方言考试、积分、游戏化学习和完整在线词典。
- AI 聊天问答、自动生成历史结论、语音克隆和合成南京话。
- 原生手机应用、小程序、电商和文创销售。
- 与南京话无明确关系的泛南京城市史、旅游、美食和景点内容。
- 使用少量语音样本推断全南京人口分布或宣称统计代表性。
- 首发英文、繁体中文或其他完整界面本地化。
- 首发批量元数据数据集、IIIF、METS、MODS 或机构级馆藏集成。
- 首发 CMS、公众后台、编辑团队权限系统和用户自助查看线索状态。
- 未经单独确认的付费文献、商业素材、专家服务、国内加速或备案服务。

## Further Notes

- 项目已建立单一领域上下文和 11 项 ADR，规格中的术语与架构选择以这些决策为准。
- 真人原创语音是正式首发的外部依赖。项目将先交付采集包；用户负责联系成年说话者并取得分项授权。录音未到位前可以部署预览，但不能宣布首发完成。
- 首轮研究必须形成带逐项来源的研究成果和材料清单，原始或下载材料与整理结果分开保存。公开内容只使用权利状态明确的媒体。
- “专家复核”不是整站发布硬门槛，但审核状态必须真实。没有专家复核的 IPA、历史语音重建或争议结论应显示待专家复核。
- 首发成功首先由元数据完整、来源可追、音频可用、无障碍通过、大陆网络稳定和纠错流程有效来判断，不设置三个月访问量目标。
- 实施开始后应先验证当前 TanStack Start 与 Cloudflare Workers 的官方部署方式，再锁定具体依赖版本；不得因框架生态偏好无差别引入 TanStack 全家桶。
